PART 1: PREPARATION (10 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

□ Have Hostinger login ready (hpanel.hostinger.com)
□ Have GitHub login ready (github.com)
□ Have text editor ready (VS Code, Sublime, Notepad++)
□ Have internet connection (at least 5 Mbps)
□ Have Node.js installed (check: node --version)
□ Bookmark these links:
   □ https://render.com/dashboard
   □ https://github.com
   □ https://hpanel.hostinger.com


PART 2: GET HOSTINGER MYSQL DETAILS (10 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

□ Login to Hostinger (hpanel.hostinger.com)

□ Navigate to: Databases > MySQL Databases

□ Find your database and note these details:
   Database Name: ____________________________________
   Username:      ____________________________________
   Password:      ____________________________________

□ Note the host (usually):
   Host: mysql.hostinger.com or mysql.cpanel.com

□ Navigate to: Databases > Remote MySQL

□ Check the status:
   □ Remote MySQL: ENABLED

□ Click "Add IP Address"
   □ Leave blank (allows all IPs, including Render)
   □ Click "Add"
   
□ You should see a confirmation message


PART 3: CREATE .ENV FILE LOCALLY (10 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

□ Open the 'backend' folder on your computer

□ Find the file: .env.example

□ Make a copy and rename it to: .env
   (Right-click > Copy > Paste > Rename to .env)

□ Open .env in your text editor

□ Find and update these lines with YOUR values:
   
   ┌─────────────────────────────────────────────────────────┐
   │ NODE_ENV=production                                     │
   │ PORT=3000                                               │
   │                                                         │
   │ DB_HOST=mysql.hostinger.com                            │
   │ DB_PORT=3306                                           │
   │ DB_USER=[PASTE YOUR USERNAME FROM HOSTINGER]            │
   │ DB_PASSWORD=[PASTE YOUR PASSWORD FROM HOSTINGER]        │
   │ DB_NAME=[PASTE YOUR DATABASE NAME FROM HOSTINGER]       │
   │                                                         │
   │ FRONTEND_URLS=https://your-hostinger-domain.com        │
   │                                                         │
   │ JWT_SECRET=[RUN THIS COMMAND, PASTE RESULT]:            │
   │   node -e "console.log(require('crypto')              │
   │   .randomBytes(32).toString('hex'))"                     │
   │                                                         │
   │ SESSION_SECRET=[RUN SAME COMMAND AGAIN, PASTE]:        │
   │                                                         │
   │ RENDER_FREE_TIER=true                                   │
   │ KEEP_ALIVE_INTERVAL=25000                              │
   │ PING_INTERVAL=30000                                     │
   └─────────────────────────────────────────────────────────┘

□ Save the file (Ctrl+S)

□ Test these changes:
   □ Open terminal in 'backend' folder
   □ Run: npm install
   □ Wait for completion (2-3 minutes)
   □ Run: npm start
   □ You should see: ✓ MySQL Database connected successfully
   □ If error → Review Hostinger credentials in .env
   □ Press Ctrl+C to stop


PART 4: COMMIT TO GITHUB (10 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

□ Open terminal in the project folder (NOT backend folder)

□ Run these commands:
   
   __________________________________

   __________________________________
   
   __________________________________

   __________________________________

□ Check GitHub website - your changes should be there


PART 5: DEPLOY TO RENDER (15 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

□ Go to: https://render.com

□ Sign up or login (free account)

□ Click: Dashboard

□ Click: New + button in top-left

□ Select: Web Service

□ If prompted, connect your GitHub account:
   □ Click "Connect Account"
   □ Authorize Render to access GitHub
   □ Return to Render

□ Select your repository:
   □ Type repository name in search
   □ Click to select it
   □ Click "Connect"

□ Configure the service:

   Service Settings:
   ┌─────────────────────────────────────────────────┐
   │ Name: lifeway-backend                           │
   │ Runtime: Node                                    │
   │ Build Command: npm install                       │
   │ Start Command: npm start                         │
   │ Branch: main                                     │
   │ Autodeploy: ON                                   │
   └─────────────────────────────────────────────────┘

□ Click Advanced:

   Environment Variables:
   ┌──────────────────────────────────────────────────┐
   │ Add each variable from your .env file:           │
   │                                                  │
   │ NODE_ENV = production                            │
   │ PORT = 3000                                       │
   │ DB_HOST = mysql.hostinger.com                    │
   │ DB_PORT = 3306                                   │
   │ DB_USER = [your username]                        │
   │ DB_PASSWORD = [your password]                    │
   │ DB_NAME = [your database]                        │
   │ FRONTEND_URLS = [your domain]                    │
   │ JWT_SECRET = [your generated secret]             │
   │ SESSION_SECRET = [your generated secret]         │
   │ RENDER_FREE_TIER = true                          │
   │ KEEP_ALIVE_INTERVAL = 25000                      │
   │ PING_INTERVAL = 30000                            │
   └──────────────────────────────────────────────────┘

□ Select Plan:
   □ Choose: Free (for testing, recommended)
   □ Or: Starter ($7/month) for always-on

□ Click: Create Web Service

□ Wait for deployment (3-5 minutes):
   □ Watch the logs
   □ Will show: Building...
   □ Then: Deploying...
   □ Finally: Build successful!

□ Get your Render URL:
   □ Copy the URL shown at top (e.g., https://lifeway.onrender.com)
   
   My Render URL: ____________________________________

□ Test the deployment:
   □ Open URL in browser: https://your-url/api/health
   □ Should see JSON response with "status": "Server is running"

□ Check the logs for success message:
   □ Should show: ✓ MySQL Database connected successfully
   □ Should show: 📚 Lifeway Backend Server running on port 3000


PART 6: UPDATE HOSTINGER FRONTEND (5 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

□ Find and open: api-service.js

□ Find line ~20 that looks like:
   const API_BASE_URL = 'http://localhost:5000/api'

□ Replace it with YOUR Render URL:
   const API_BASE_URL = 'https://your-app.onrender.com/api'
   
   My specific line:
   const API_BASE_URL = '__________________________________'

□ Save the file (Ctrl+S)

□ Upload to Hostinger using one of these methods:

   METHOD A: Using cPanel File Manager
   ┌──────────────────────────────────────────────────────┐
   │ 1. Login to cPanel (hpanel.hostinger.com)            │
   │ 2. Click: File Manager                               │
   │ 3. Go to: public_html folder                          │
   │ 4. Right-click > Upload                              │
   │ 5. Select api-service.js                              │
   │ 6. Upload complete!                                   │
   └──────────────────────────────────────────────────────┘

   METHOD B: Using FTP (if you know how)
   ┌──────────────────────────────────────────────────────┐
   │ 1. Open FTP client (FileZilla, etc)                  │
   │ 2. Connect to your Hostinger FTP                      │
   │ 3. Navigate to public_html                            │
   │ 4. Drag api-service.js into public_html               │
   │ 5. Upload complete!                                   │
   └──────────────────────────────────────────────────────┘

□ Clear your browser cache:
   □ Press: Ctrl + Shift + Delete
   □ Click: Clear browsing data
   □ Click: Clear data


PART 7: TEST EVERYTHING (10 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

□ Open your Hostinger website in a fresh browser window

□ Open DevTools (F12 on keyboard)

□ Go to Console tab

□ Try a login attempt (use test credentials if available)

□ In Console, should see API URL:
   □ Shows something like: Fetch to https://your-app.onrender.com/api/...

□ Check Network tab:
   □ Click any network request
   □ Headers tab should show your Render backend URL
   □ Status should be 200 (success) or 401 (need better credentials)

□ Test some functionality:
   □ Try login form
   □ Try contact form
   □ Try viewing courses
   □ Try navigating to different pages

□ No CORS errors should appear:
   □ If you see "CORS error":
      - Go back to Part 5
      - Check FRONTEND_URLS is correct
      - Redeploy from Render dashboard

□ Watch Render logs for any errors:
   □ Render Dashboard > Your Service > Logs
   □ Should see requests coming in
   □ Should see "POST /api/..." with status 200


PART 8: VERIFICATION (5 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

SUCCESS INDICATORS - Check if you see these:

Backend Indicators:
□ Render dashboard shows GREEN status (not orange/red)
□ Logs show: "✓ MySQL Database connected successfully"
□ Logs show: "📚 Lifeway Backend Server running on port 3000"
□ Health check works: https://your-url/api/health returns JSON

Frontend Indicators:
□ Browser opens your Hostinger site without 404 errors
□ Browser console is CLEAN (no red error messages)
□ Network tab shows requests to Render URL (not localhost)
□ At least one API call succeeds (status 200 or 2xx)

Database Indicators:
□ Login form sends request (Network tab shows POST)
□ Form submission creates data (check Hostinger database in cPanel)
□ Dashboard pages load and show data

Functionality Indicators:
□ Student login works (or shows authentication error, meaning backend works)
□ Forms don't timeout (no "Request timeout" errors)
□ No CORS errors in console
□ All pages load without 500 errors


PART 9: OPTIONAL - PRODUCTION OPTIMIZATION (15 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

Only do this if you want always-on production:

□ Upgrade Render plan from Free to Starter ($7/month):
   □ Render Dashboard > Your Service > Plan
   □ Select Starter Plan ($7/month)
   □ Provide payment method
   □ Upgrade
   □ App will always stay running (even when idle)

□ Enable error tracking (optional):
   □ Create Sentry account (sentry.io)
   □ Get DSN (unique ID)
   □ Add to Render environment variables:
      SENTRY_DSN = [your DSN]
   □ Redeploy
   □ Now you'll be alerted of any errors

□ Set up database backups:
   □ Hostinger cPanel > Databases > Backups
   □ Schedule monthly automatic backups
   □ Download one backup to test it works

TROUBLESHOOTING QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════════

PROBLEM: "Cannot connect to backend"

SOLUTION:
┌─────────────────────────────────────────────────────────────┐
│ 1. Check Render URL is correct (includes /api)              │
│ 2. Test URL in browser: https://your-url/api/health         │
│ 3. Check Render logs for errors                              │
│ 4. Verify all environment variables are set                  │
│ 5. Make sure MySQL credentials are correct                   │
└─────────────────────────────────────────────────────────────┘


PROBLEM: "MySQL connection failed"

SOLUTION:
┌─────────────────────────────────────────────────────────────┐
│ 1. Go to Hostinger cPanel                                    │
│ 2. Databases > Remote MySQL                                  │
│ 3. Ensure it shows "Enabled"                                 │
│ 4. Ensure IP address field is blank (%) to allow all         │
│ 5. Verify username, password, database name in .env          │
│ 6. Try test locally first: npm start in backend              │
└─────────────────────────────────────────────────────────────┘


PROBLEM: "CORS error - blocked"

SOLUTION:
┌─────────────────────────────────────────────────────────────┐
│ 1. Go to Render Dashboard                                    │
│ 2. Click your service                                        │
│ 3. Click "Environment"                                       │
│ 4. Find FRONTEND_URLS                                        │
│ 5. Make sure it has YOUR Hostinger domain exactly            │
│ 6. Save and redeploy                                         │
│ 7. Clear browser cache (Ctrl+Shift+Del)                      │
└─────────────────────────────────────────────────────────────┘


PROBLEM: "App spins down / becomes slow"

NORMAL ON FREE TIER! (This is expected)

SOLUTION:
┌─────────────────────────────────────────────────────────────┐
│ Option A: Upgrade to Starter plan ($7/month) for always-on   │
│ Option B: Accept slow startup after inactivity (it wakes up) │
│ Option C: Add cron job to ping /api/health every 10 min      │
└─────────────────────────────────────────────────────────────┘
